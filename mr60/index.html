Overview

MR60BHA2 mmWave 센서 마스터 클래스

기본 번들 제공되는 XIAO ESP32C6와 호흡/심박 레이더 모듈을 마스터하기 위한 하드웨어 연동 허브입니다.

공식 번들 MCU: Seeed Studio XIAO ESP32C6

이 센서 키트는 초소형 폼팩터에 차세대 Wi-Fi 6, Bluetooth 5, Zigbee / Thread (802.15.4) 무선 스택을 모두 통합한 XIAO ESP32C6 개발 보드가 결합되어 출시됩니다.

RISC-V 싱글 코어

160MHz 클럭의 효율적인 RISC-V 아키텍처로 레이더 데이터 파싱 및 무선 IoT 전송을 동시 수행합니다.

스마트홈 최적 프로토콜

Zigbee 및 Thread 연동을 지원하여 가구 밀착형 인체 감지 센서 네트워크 구축에 가장 앞서 있습니다.

Grove 에코시스템

키트 메인보드 상의 Grove 커넥터를 통해 별도의 복잡한 납땜 없이 바로 UART 시리얼 데이터 전송을 시작합니다.

센서 주요 특징

MR60BHA2는 60GHz FMCW(주파수 변조 연속파) 레이더 기술을 활용하여 인체 접촉 없이 미세한 흉부 움직임을 감지하고, 이를 분석하여 호흡 및 심박 수치를 출력하는 고정밀 센서 모듈입니다.

  • 주파수 대역 58GHz - 62GHz (60GHz 대역)
  • 호흡 감지 범위 0.4m ~ 1.5m (최적의 정밀 구역)
  • 움직임 감지 범위 최대 6.0m (존재 감지)
  • 통신 인터페이스 UART (TTL 3.3V, 115200 bps)
  • 공급 전원 4.6V ~ 6.0V (정격 5.0V / 1A 이상 권장)

센서 도착 후 준비 단계

센서를 개봉하고 본격적인 개발에 들어가기 전에 다음 단계를 확인하세요.

  • 안정적인 5V 외부 전원 공급 장치 확보

    레이더 센서는 무선 신호 송출 시 최대 200mA 이상의 피크 전류를 소비하므로, MCU의 5V 핀 전원 대신 별도의 전용 어댑터를 사용하는 것이 동작 오류를 막습니다.

  • USB to UART 컨버터 (FTDI 등) 준비

    아두이노 소스 코드를 올리기 전, PC와 센서를 직접 연결하여 Seeed의 공식 mmWave 디버깅 소프트웨어를 통해 센서가 정상 동작하는지 테스트해보는 것이 좋습니다.

  • 센서 배치용 고정 구조물 브래킷 또는 삼각대

    미세한 호흡/심박수를 측정하기 때문에 센서 껍데기가 흔들리면 측정에 큰 오차가 발생합니다. 단단하게 고정할 스탠드나 브래킷이 필수적입니다.

MR60BHA2 센서 작동 메커니즘 및 실무 연동 절차

이 섹션은 하드웨어 엔지니어 및 소프트웨어 개발자가 MR60BHA2 센서를 실제 상용 제품이나 스마트홈 시스템에 이식할 때 반드시 숙지해야 하는 **동작 상태 상태도(State Machine)**와 **표준 개발 절차**, 그리고 **실무 기하학적 제약 조건**을 이론적으로 다룹니다.

A. 센서 내부의 동작 상태 머신 (State Machine)

센서는 전원이 켜진 후, 수신 전파의 실시간 변화 정도에 따라 내부 알고리즘이 4가지 상태를 자동 천이하며 동작합니다. 각 상태의 트리거 요건과 동작 특성을 파악해야 오동작 없는 소프트웨어 기획이 가능합니다.

  • 비재실 (Vacant / No one): 감지 영역 내에 어떠한 전파 반사 변화(움직임 및 위상 변화)도 없을 때의 대기 상태입니다. 센서는 이 상태에서 배경 잡음(Background Noise)을 지속 샘플링하여 클러터 맵을 갱신합니다.
  • 동적 존재 (Active / Movement): 영역 내로 인체가 진입하거나 큰 몸짓(걷기, 손 흔들기 등)이 감지되는 순간 즉각 진입합니다. 이 상태에서는 생체 신호(호흡/심박) 값의 신뢰도가 낮아지며, 재실 판정 플래그가 우선 활성화됩니다.
  • 정적 존재 (Static / Breathing): 대상이 자리에 가만히 앉거나 누워 미세한 움직임만 취할 때 천이됩니다. 내부 DSP 파이프라인이 가동되어 흉부의 미세 위상차 분석(호흡수 및 심박수)을 정밀 추출하는 주동작 상태입니다.
  • 수면 분석 (Sleep State): 정적 존재 상태가 장시간 유지되며 호흡의 주기성과 미세 움직임의 빈도가 낮아지면 진입합니다. 센서는 수면 깊이(깊은 수면, 얕은 수면, 무호흡 상태)를 패턴 인식하여 별도의 수면 통계 패킷을 내보냅니다.

B. 센서 연동 표준 개발 절차 (Standard Procedure)

MCU(XIAO ESP32C6 등) 소스코드 작성 시 아래의 단계적 절차를 순서대로 구현해 주어야 센서의 성능을 최대로 발휘할 수 있습니다.

  1. 1. 초기 파워온 안정화 대기 (Initialization Delay):
    센서 내부 알고리즘이 부팅 후 주변 환경의 정적 클러터를 분석하고 자가 진단(Self-Test)을 마치는 데 약 **5~10초**의 시간이 필요합니다. 이 시간 동안 들어오는 시리얼 데이터는 무시하거나 보정 시간으로 힙에 쌓아둡니다.
  2. 2. 감도 및 매개변수 설정 (Configuration Write):
    설치된 환경의 방 크기나 노이즈 유무에 따라, MCU가 센서로 **감도 제어 커맨드(Sensitivity Level)**나 **감지 범위 경계(Range Gates)** 설정 패킷을 송신하여 하드웨어를 튜닝합니다.
  3. 3. 링버퍼(Ring Buffer) 기반 수신 및 헤더 싱크 동기화:
    센서는 끊임없이 바이트 스트림을 송출합니다. MCU는 시리얼 수신 인터럽트에서 데이터를 링버퍼에 저장하고, 순차적으로 `0x53 0x59` 헤더를 찾아내 프레임의 시작점을 실시간으로 동기화해야 합니다.
  4. 4. 체크섬 검증 및 데이터 언패킹 (Checksum & Parsing):
    수신된 프레임의 전체 바이트 합을 계산하여 체크섬 바이트와 일치하는지 무결성을 검증합니다. 일치할 경우에만 제어 단어(Control Word)와 명령 단어(Command Word)를 매칭하여 실제 데이터 영역(Data Payload)의 생체 신호 수치를 변수에 대입합니다.
  5. 5. 예외 및 아웃라이어 필터링 (Software Filtering):
    대상이 수면 중 자세를 뒤척이거나 감지 범위를 일시적으로 이탈할 때, 센서에서 호흡수나 심박수가 순간적으로 `0`으로 드롭되는 아웃라이어가 발생할 수 있습니다. MCU 펌웨어 단에서 **이동 평균 필터(Moving Average)**나 **직전 유효값 유지(Hold Filter)** 알고리즘을 추가하여 연동 데이터의 품질을 안정화하십시오.

C. 실무 설치 기하학 및 물리적 제약 이론

설치 높이와 각도 조건

* **벽면 장착 (Wall Mount):** 바닥에서 1.2m ~ 1.5m 높이에 설치하고, 센서 전면이 사람의 가슴을 향해 대각선 하향(45도 내외)으로 조준되도록 조절합니다.
* **천장 장착 (Ceiling Mount):** 침대 머리맡 바로 위 천장(높이 2.5m 내외)에 수직 방향으로 설치하여 흉부의 팽창 운동을 직각 방향에서 최대로 샘플링할 수 있도록 포지셔닝합니다.

전파 간섭 회피 조건

* **기계적 진동 배제:** 레이더는 미세 진동을 인체 생체 신호로 오인하므로 에어컨, 선풍기 날개, 흔들리는 화초, 물이 흐르는 금속 배관 등 주기적인 미세 변위가 발생하는 대상이 레이더 안테나 빔 각도(FOV) 정면에 위치해서는 안 됩니다.
* **투과 및 난반사:** 60GHz 전파는 목재, 유리, 석고보드를 일부 투과하므로 벽 너머에 있는 사람의 움직임이 노이즈로 잡힐 수 있습니다. 감도 레지스터를 활용해 거리를 차단하십시오.

실무 활용 시나리오 예시

* **독거노인 위급 감지 (Sleep Apnea):** 수면 중 호흡수가 0인 상태(무호흡)가 15초 이상 유지되고, 움직임 강도가 없을 때 MCU가 Wi-Fi 6망을 통해 보호자나 관제 센터로 비상 긴급 패킷을 전송하는 경보 시스템 기획이 가능합니다.
* **진정한 정적 재실 조명 제어:** 기존 적외선(PIR) 센서와 달리 가만히 앉아서 독서나 컴퓨터 업무를 하는 정적 상태에서도 재실을 100% 감지하므로, 화장실이나 서재에서 사람이 가만히 있을 때 전등이 갑자기 꺼지는 고질적인 UX 결함을 해결할 수 있습니다.

D. MCU 물리 결선 및 펌웨어 업로드 가이드

1. 공통 그라운드(Common Ground) 및 외부 전원

레이더 센서를 구동할 때, 노이즈와 리셋 루프를 방지하기 위해 5V 외부 전용 어댑터를 사용하는 것이 권장됩니다.

이때, **외부 전원의 그라운드(GND)와 ESP32/XIAO 보드의 그라운드(GND)는 반드시 하나로 묶여야(Common Ground)** 합니다. 그라운드를 공통으로 연결하지 않으면 두 기기 간의 통신 기준 전위가 달라져 UART 데이터가 전부 깨지게 됩니다.

2. 업로드 시 핀 충돌 회피 (중요 트러블슈팅)

* **기본 포트 충돌:** 아두이노 Uno나 일부 ESP32 보드에서 기본 시리얼 핀(RX0, TX0)에 레이더 센서를 상시 연결해두면, USB를 통한 **PC의 코드 업로드 신호와 센서의 송출 전파 신호가 충돌**하여 업로드 에러가 납니다.

* **대책:** 코드 업로드 중에는 센서의 RX/TX 선을 일시적으로 뽑거나, 본 가이드의 기본값처럼 **XIAO ESP32C6의 Hardware Serial 1(GPIO 20/21) 포트**나 **ESP32 DevKit의 Hardware Serial 2(GPIO 16/17) 포트**로 물리 핀을 분리하여 설계하십시오.

E. 라즈베리 파이 5 (8GB) 기반 Edge AI 시스템 아키텍처 및 무선 데이터 전송

MCU(XIAO ESP32C6)가 레이더 센서로부터 디코딩한 생체 데이터를 동일 네트워크 상의 라즈베리 파이 5 (8GB) 중앙 서버로 쏘아보내고, 파이 5의 강력한 하드웨어 자원을 활용해 실시간 Edge AI 추론(수면 무호흡 판정, 낙상 감지 등)을 구동하는 전체 시스템 아키텍처 가이드입니다.

                                                          | (Wi-Fi 송신)
                                                          v
                                                 [공유기 (AP Router)]
                                                          |
                                                          v
                                          [라즈베리 파이 5 (Edge AI Server)]
                                          - MQTT Broker (데이터 중계)
                                          - Python Engine (시계열 모델 추론)
                                          - Web Dashboard (실시간 시각화)
1. MQTT 방식 (실시간 Edge AI에 강력 권장)

라즈파이 5에 MQTT Broker를 켜두고, ESP32가 파이의 IP로 JSON을 쏘아 보냅니다. 오버헤드가 적어 수백 ms 단위의 실시간 데이터 파이프라인 구축에 최고입니다.

라즈베리 파이 5 Broker 세팅: 
# Mosquitto Broker 및 클라이언트 설치
sudo apt update
sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

* **연동 원리:** ESP32가 와이파이를 통해 `radar/vital_signs` 토픽으로 JSON 바이너리를 발행하면, 라즈베리 파이 5의 파이썬 프로세스가 이를 실시간으로 구독해 추론에 사용합니다.

2. 파이 5(8GB) 수신용 파이썬 Edge AI 엔진 코드

파이에서 실행되어 ESP32가 보낸 실시간 생체 수치들을 시계열로 누적하고 머신러닝/AI 추론(예: TFLite, ONNX)을 구동하는 데몬 스크립트입니다.

라즈파이 Python AI 엔진 예제 (inference.py): 
import json
import paho.mqtt.client as mqtt
# import tflite_runtime.interpreter as tflite # AI 모델 라이브러리 예시

# 시계열 추론을 위해 최근 30개의 수치 수집용 버퍼
vital_history = []

def run_edge_ai_inference(history):
    # 예시: 30초 분량 데이터가 다 차면 AI 추론 가동
    # input_data = np.array(history)
    # prediction = interpreter.predict(input_data)
    print(f"▶ [AI Engine] 최근 시계열 분석 중... 데이터 버퍼 크기: {len(history)}")
    
    # 임계 검출 및 패턴 인식 로직 수행
    last_breath = history[-1]['breath_rate']
    if last_breath == 0:
        print("🚨 [WARNING] 무호흡 의심 상태 감지! 비상 경보 트리거!")

def on_message(client, userdata, msg):
    payload = json.loads(msg.payload.decode())
    vital_history.append(payload)
    if len(vital_history) > 30:
        vital_history.pop(0)
    
    print(f"★ [수신] 심박: {payload['heart_rate']} bpm | 호흡: {payload['breath_rate']} rpm")
    if len(vital_history) >= 10:
        run_edge_ai_inference(vital_history)

client = mqtt.Client()
client.on_message = on_message
client.connect("localhost", 1883, 60)
client.subscribe("radar/vital_signs")
client.loop_forever()

F. Seeed Studio 공식 디버깅 소프트웨어 및 수동 튜닝 가이드

MR60BHA2 센서의 감도를 조절하거나 정적 환경(가구, 장애물)의 반사파를 소거하고 감지 경계를 설정하기 위해, Seeed Studio는 공식 **상위 컴퓨터 디버깅 툴 (mmWave Radar Upper Computer Software)**을 제공합니다.

1. PC 및 센서 연결 방식

MCU 보드 없이 센서 자체를 직접 PC에 연결하여 환경 설정을 바이트 명령어로 전송하거나 실시간 파형을 확인해야 합니다.

* **준비물:** USB to UART (TTL 3.3V) 컨버터 (예: FTDI, CP2102 칩셋 컨버터)
* **결선 매핑:** 컨버터 5V → 센서 VCC, 컨버터 GND → 센서 GND, 컨버터 RX → 센서 TXD, 컨버터 TX → 센서 RXD
* **연결 설정:** Seeed 위키(Wiki)에서 각 모듈용 'Upper Computer' 실행 파일을 다운로드하고, 포트 번호를 지정한 뒤 통신 속도를 **115200 bps**로 지정하여 오픈합니다.

2. 상위 컴퓨터 툴의 핵심 튜닝 기능
  • 자가 캘리브레이션 (Static Suppress / Clutter Calibration):
    빈 방에 센서를 설치한 상태에서 이 명령을 실행하면, 1~2분간 가구, 침대 프레임, 벽면 등 움직이지 않는 구조물의 고정 반사파 세기를 측정하여 내부 메모리에 **정적 노이즈 맵(Clutter Map)**으로 자동 박아둡니다. 이후 인체 감지 시 이 Clutter Map을 빼고 연산하므로 정적 가구로 인한 오동작이 원천 차단됩니다.
  • 감지 거리 게이트 범위 조정 (Range Limit Gate):
    센서 전방의 감지 시작 경계(Min Distance)와 차단 경계(Max Distance)를 미터 단위로 세팅합니다. 예컨대 침실 수면 모니터링 시 범위를 딱 `0.4m ~ 1.8m`로 잠가두면, 침대 밖 문을 열고 지나가는 다른 사람이나 가전제품 동작을 완전히 무시할 수 있습니다.
  • 민감도 임계값(Sensitivity Threshold) 세팅:
    방안에 바람에 흔들리는 커튼이나 화초가 있는 경우, 존재 감도 단계를 1~10 중 낮추어(예: Level 3) 미세 바람 노이즈를 컷오프(Cut-off)하고 인간만 정밀하게 검출하도록 임계값을 조절합니다.

Deployment Guide

현장 설치 가이드 및 문제해결

60GHz mmWave 레이더 센서의 뛰어난 감도 특성으로 발생할 수 있는 노이즈 문제와 하드웨어 트러블슈팅 요령입니다.

이상적인 센서 설치 요령

mmWave 센서는 측정 빔 각도 안에 진동하는 물체가 있을 시 인체로 오인하기 매우 쉽습니다. 최적의 감지 효율을 위해 다음 설치 배치를 준수하세요.

침대 헤드 부분 위에서 아래로 사선 배치 (추천)

침대에 누워 잠자는 사람의 호흡을 체크하고자 할 때 가장 오차가 적은 배치입니다. 누워 있는 상태에서 레이더 빔이 흉부를 수직으로 잘 관통하도록 침대 머리맡 벽에서 약 45도 기울여 하향 설치합니다.

지면으로부터 1.0m ~ 1.5m 높이 고정

센서 높이가 사람의 심장 및 가슴 높이와 비슷하게 위치할수록 측정 정밀도가 극대화됩니다.

강력한 지지대 고정

센서를 옷걸이, 약한 플라스틱 바 등에 얹어두면 센서 자체 진동 및 주변 바람에 센서 프레임이 흔들려 실시간 호흡수가 심하게 왜곡(예: 0회 혹은 비정상적인 폭증)됩니다.

대표적인 문제 해결 (Q&A)

Q1. 방안에 사람이 없는데 '사람 존재' 및 호흡수가 찍힙니다.

원인: mmWave 빔 감지 범위 내에 회전하는 선풍기 날개, 펄럭이는 커튼, 에어컨 바람에 흔들리는 실내 식물이 있을 가능성이 90% 이상입니다.
해결책: 센서 설치 각도를 미세하게 조정하여 움직이는 기계 및 흔들리는 물체 방향을 회피하거나, 센서 감도(Sensitivity)를 하향 설정(Level 1)하여 불필요한 신호를 컷오프하십시오.

Q2. 호흡 수치와 심박수가 정기적으로 0으로 뚝뚝 떨어집니다.

원인: 측정 유효 범위(1.5m 이내)를 벗어났거나, 사람이 너무 격렬하게 움직이고 있는 중일 수 있습니다. 센서는 대상이 가만히 안정 상태일 때(Static) 미세 흉부 팽창을 정확하게 샘플링합니다.
해결책: 대상자의 부동 수면 자세를 확보하거나, 센서 거리를 침대로부터 1m 이내로 더 가깝게 전진 설치하십시오.

Q3. 시리얼 모니터에 깨진 문자가 출력되거나 데이터 수신이 아예 안 됩니다.

원인: 보드 레이트(Baud Rate) 미스매치 또는 아두이노 보드의 하드웨어 시리얼 핀 크로스 배선 오작동입니다.
해결책: 센서의 기본 통신 속도는 115200bps 입니다. 9600bps로 연결되어 있지 않은지 아두이노 코드를 검토하시고, TX핀과 RX핀을 교차 연결(센서 TX -> MCU RX)했는지 반드시 손으로 재확인해 주십시오.

Core Engineering Theory

60GHz FMCW 레이더 핵심 이론 & 신호 처리

MR60BHA2 센서의 내부 작동 원리인 FMCW 주파수 변조 연속파 기술과 호흡/심박 위상 검출 알고리즘을 깊이 있게 학습합니다.

1. FMCW 레이더 및 거리 계측 원리

FMCW(Frequency Modulated Continuous Wave) 레이더는 시간에 따라 주파수가 선형적으로 상승하는 **칩(Chirp) 신호**를 지속해서 방사합니다.

거리 계산 공식 (Distance Formula)
R = (c × f_b × T_c) / (2 × B)
  • c: 전파 속도 (빛의 속도, 3 × 10⁸ m/s)
  • f_b (Beat Frequency): 송수신 주파수 차이 (비트 주파수)
  • T_c (Chirp Duration): 주파수 쓸기(Sweep) 주기
  • B (Bandwidth): 스윕 대역폭 (주파수 범위)

왜 60GHz 레이더가 24GHz보다 우수할까요?

레이더의 거리 해상도(Range Resolution)는 d_R = c / (2 × B)로 결정됩니다.

  • 24GHz 레이더 (대역폭 250MHz 내외): 물리적인 거리 분해능이 약 60cm에 달합니다. 사람과 주변 사물을 정밀하게 발라내지 못합니다.
  • 60GHz 레이더 (대역폭 최대 4GHz): 거리 분해능이 3.75cm 수준으로 극히 정밀합니다. 침대 위의 인간의 신체와 매트리스, 주변 정적 장애물을 명확히 구분할 수 있습니다.

2. 미세 도플러(Micro-Doppler) 및 생체 센싱

호흡이나 심장 박동은 mm 단위 이하의 미세한 변위를 발생시키기 때문에 단순한 거리(R) 분석으로는 측정이 어렵습니다. 대신 레이더는 대상체로부터 반사된 전파의 **위상(Phase, θ) 변화**를 계측합니다.

변위에 따른 위상 변위 공식
Δθ = (4 × π × Δd) / λ
  • Δθ (Delta Phase): 수신 신호의 위상 변화량 (라디안)
  • Δd (Displacement): 흉부의 물리적 변위 (호흡: 4~12mm, 심박: 0.1~0.5mm)
  • λ (Wavelength): 전파의 파장 (60GHz = 5mm, 24GHz = 12.5mm)

파장(λ)의 차이가 만드는 민감도의 차이

60GHz 주파수 대역의 파장(λ)은 단 **5mm**에 불과합니다. 전파의 왕복 특성 때문에 호흡에 의해 가슴이 **1.25mm**만 움직여도 수신 위상은 무려 **180도(π rad, 반 파장 왕복)** 만큼 완전히 뒤집힙니다. 심장 박동에 의한 **0.25mm**의 미세 변위에도 위상이 **36도** 변하여, 24GHz 레이더(14.4도 변화) 대비 위상 감도가 **약 2.5배 민감**하므로 노이즈 속에서도 심박 변형 신호를 정확히 살려낼 수 있습니다.

3. 내부 DSP 신호 처리 파이프라인 (Signal Processing Flow)

레이더 전면 안테나를 통해 수신된 고주파 아날로그 반사파가 수치 데이터로 변환되는 5단계 필터 아키텍처입니다.

1단계: Range FFT
수신된 아날로그 비트 주파수(Beat frequency)를 디지털로 ADC 샘플링한 후 고속 푸리에 변환(FFT)을 수행합니다. 이를 통해 신호 강도를 거리 영역별(Range Bin)로 주파수 스펙트럼 화 합니다.
2단계: Target Bin 위상 추출
여러 거리 구간 중 인체가 검출된 특정 거리 빈(Target Range Bin)을 선정하고, 해당 위치에서의 복소수 신호의 위상각(Phase Angle) 데이터를 시간에 따라 지속해서 아카이빙합니다(Slow-time sampling).
3단계: 위상 보정 (Unwrap)
위상은 -π에서 +π 범위로 제한되므로, 범위를 넘을 때 튀는 현상을 방지하기 위해 연속된 복소 위상 값을 펴주는 Unwrap 연산을 적용합니다. 이어 전위차 성분을 활용하여 인접 프레임 간 차분을 구합니다.
4단계: 대역 필터링 (BPF)
추출된 미세 변위 신호에서 각각의 타깃 신호를 분리하기 위해 디지털 Bandpass Filter(대역통과필터)를 통과시킵니다.
  • 호흡 신호 필터 (Respiration Filter): 0.1Hz ~ 0.5Hz (분당 6회 ~ 30회 감지 영역)
  • 심박 신호 필터 (Heartrate Filter): 0.8Hz ~ 2.5Hz (분당 48회 ~ 150회 감지 영역)
5단계: 주파수 스펙트럼 추정
필터링된 호흡/심박 변형 데이터에 대해 2차 FFT 연산 또는 웰치 메서드(Welch's Method) 스펙트럼 분석을 수행하여 가장 지배적인 주파수 피크(Peak)를 검출하고, 이 주파수 성분을 바탕으로 최종 분당 횟수(BPM)를 결정하여 UART 프레임에 패킹합니다.

4. 레이더 정적 클러터 및 예외 처리 (Clutter Suppression)

정적 클러터 소거 (Static Clutter Suppression)

실내에는 벽, 침대 헤드, 탁자 등 전파를 매우 강하게 반사하는 금속 및 콘크리트 구조물이 존재합니다. 레이더는 **동적 대상 지시(MTI - Moving Target Indicator)** 기술을 탑재하여 시간에 따라 신호가 전혀 변하지 않는 정적 클러터 반사파 성분을 수학적으로 완전 차감하여 대상의 순수한 움직임 정보만을 발라냅니다.

동적 노이즈 (Dynamic Interference) 대책

선풍기 날개, 커튼의 흔들림, 에어컨의 바람 등은 mmWave 대역에서 가짜 생체 신호(False Presence)를 형성할 위험이 큽니다. 이를 방지하기 위해 MR60BHA2 센서는 **기본 감도 임계값(Threshold Gate)** 조절 레지스터를 갖고 있으며, 1차 주파수 강도가 특정 dB 이상일 때만 생체 분석 엔진을 구동하는 방식으로 허위 동작을 필터링합니다.
